# MEDOED

EDR система, которую я написал для мониторинга рабочих станций в сети. Идея простая: один сервер собирает телеметрию с агентов и помогает вовремя заметить подозрительную активность.

Пока умеет не так много, но база уже работает: агенты регистрируются, шлют heartbeat, собирают информацию о процессах и отправляют события на сервер. Дальше буду добавлять детекцию угроз и автоматические алерты.

## Что внутри

Проект разделён на две части:
- **Agent** — C++ приложение, которое ставится на рабочие станции и собирает данные
- **Backend** — Go сервер, который принимает данные от агентов и хранит их в PostgreSQL

## Быстрый старт

Запуск через Docker:

```bash
docker-compose up -d
```

Бэкенд на `http://localhost:8080`. Проверка через curl:

```bash
curl http://localhost:8080/health
```

## Сборка вручную

### Backend

```bash
cd src/backend
go mod download
go build -o server .
./server
```

Для работы нужен PostgreSQL. Настройки подключения берутся из `config.json`, но можно переопределить через переменные окружения.

### Agent

```bash
cd src/agent
mkdir -p build && cd build
cmake .. && make
./agent
```

Агент сначала попробует прочитать конфиг из файла, если его нет — возьмёт значения по умолчанию или переменную `BACKEND_URL`.

## Настройка

Конфиги лежат рядом с кодом, их легко править. Для бэкенда — `src/backend/config.json`, для агента — `src/agent/config.json`.

Для агента важно указать правильный `backend_url`. Если запускаете через Docker в одной сети — ставьте `http://backend:8080`. Если агенты будут стучаться на публичный IP — соответственно IP или домен.

```json
{
  "backend_url": "http://192.168.1.100:8080",
  "heartbeat_interval": 30
}
```

`heartbeat_interval` — как часто агент будет отправлять сигнал о жизни. По умолчанию 30 секунд, для слабых машин можно увеличить.

## Как развернуть в сети организации

1. На сервере PostgreSQL и бэкенд. Можно через Docker, можно вручную — без разницы.
2. На каждую рабочую станцию ставим агента. Конфиг с адресом сервера кладём в `/etc/medoed/agent.json` или передаём через `AGENT_CONFIG_PATH`.
3. Агенты сами регистрируются и начинают слать данные.

Никаких сложных настроек, всё работает из коробки. Пока нет авторизации и TLS, но для тестирования/локальной сети сойдёт.

## API

Основные эндпоинты:

- `POST /api/v1/agents/register` — зарегистрировать агента
- `POST /api/v1/agents/heartbeat` — heartbeat от агента
- `GET /api/v1/agents` — список агентов
- `POST /api/v1/events` — отправить событие
- `GET /api/v1/events` — события
- `GET /api/v1/alerts` — алерты

## Требования

- Go 1.21+ для бэкенда
- C++17, libcurl, CMake 3.10+ для агента
- PostgreSQL 16+ для хранения данных
